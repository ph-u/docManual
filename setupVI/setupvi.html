VIM packages (~/.vim/plugin/*.vim)

  • automatic bracket pairing
  • pop-up tab for similar text in vi/vim insert mode
    • During inserting text, press `tab` key to call a pop-up tab listing all textstrings (identified by space-separation)
    • Use `enter` key to select the chosen textstring
    • To use the ordinary `tab` spacing function, a space needs to be explicitly inserted before pressing `tab`

Explanation

In vi/vim normal mode, you can type ":" to indicate the line being typed is a command OR "/" to indicate the line being typed is a plain textstring to be searched. In .vimrc file each line is a command, hence no need to include the ":"

  • syntax on: command colour within vi/vim (according to the file extension)
  • set number: show line number within vi/vim (opposite command: `set nonumber`)
  • set hlsearch
    • In normal mode use `/[text]` to search the textstring [text] within the vi/vim editor window section(s)
    • Show (yellow) highlight withinin vi/vim search (use noh to revert highlighting after search)
  • set ts=8
    • ts = tab space
    • Every tab has a standard ts value in vi/vim and is converted into spaces when saved, which can cause conflict between IDE/text editors
    • A vi/vim ts has a max value of 8 spaces; ts=4 means 2 spaces; ts=2 means 4 spaces; ts=8 means 1 tab character
  • set smartindent: automatic indent one ts unit for newlines within a pair of brackets - ( ) OR [ ] OR { }
  • bel (vert) term
    • bel: below; term: terminal -- this command would initiate a plain terminal below the vi/vim editor
    • An alternative is add vert (vertical) in the command (before/after the bel) -- the combination would add a terminal at the right hand side of the active vi/vim editor section

Multiple Files in the Same Window

Sectioning can be called in vi/vim normal mode. Same/Different files can be opened within the same window at the same time. Commands are functional within the same window across all editor sections. All sections are always equal in width for vertical sectioning and equal in height for horizontal sectioning. Each window (across sections) only has one editing point and all other sections can be read as plain text.
